# ElevenLabs TTS Integration

## Context

The `elevenlabs-tts` branch exists to add ElevenLabs as a third TTS engine alongside macOS `say` and OpenAI. PLAN.md already captures user decisions: use Node.js + official `@elevenlabs/elevenlabs-js` SDK, keep section-by-section narration, ElevenLabs is opt-in via `DIFF_REVIEW_TTS_ENGINE=elevenlabs`.

## Files to Create

### 1. `package.json`
Minimal Node.js manifest with `@elevenlabs/elevenlabs-js` dependency. Run `npm install`.

### 2. `scripts/tts-elevenlabs.js`
Standalone Node.js CLI (~90 lines) that:
- Parses args: `--voice <id>`, `--speed <float>`, `--model <id>`, `--debug`; remaining args or stdin = text
- Inits `ElevenLabsClient` with `ELEVENLABS_API_KEY`
- Calls `textToSpeech.convert()` with `outputFormat: "mp3_44100_128"`
- Collects stream to buffer (handle both Node Readable and Web ReadableStream)
- Writes temp mp3, spawns `afplay` detached, cleans up after playback
- Exits non-zero on failure (so tts.sh falls back)
- Defaults: voice from `ELEVENLABS_VOICE_ID` env or hardcoded `pjcYQlDFKMbcOUp6F5GD`, model `eleven_turbo_v2_5`, speed `1.0`
