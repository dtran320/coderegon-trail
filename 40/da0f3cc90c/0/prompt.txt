# Plan: PR Comprehension Game via O'Reorg Trail

## Context

The O'Reorg Trail game (`/fly-visual`) currently teaches web framework request pipelines — you travel 8 stops representing pipeline stages (middleware, router, controller, etc.) and answer quiz questions along the way. But the game format is perfect for PR comprehension too: travel through the PR's change groups, answer questions about what changed and why.

The goal is to extend `/fly-visual` so it accepts PR inputs (PR number, URL, git range) and generates a PR-specific trail game. The game engine and HTML generation stay the same — only the trail data source changes.

## Approach: Extend fly-visual + New Agent

**3 files touched:**

### 1. Create `agents/pr-trail-extractor.md` (new agent)

A sonnet-model agent that takes a PR source and produces TRAIL_DATA JSON. Flow:

1. **Fetch PR** — `gh pr diff <N>` + `gh pr view <N>` (or `git diff <range>` for branches)
2. **Group changes** — Feature-first grouping (related files together), then layer fallback. Rank each group 1-7 by importance.
3. **Build 5-8 stops** — Each change group becomes a stop, ordered most-important-first. Picks the "money" code snippet (15-25 lines of the actual file, not diff format) via Read tool.
4. **Derive 4 party members** — Key domains/layers touched by the PR (e.g., "API Layer", "Data Model", "Business Logic", "Test Coverage")
5. **Generate quiz events** — 1 event per stop, mixing types (weather/river/encounter/misfortune). Questions test understanding of the changes (why/what-if/trace/connect/spot-the-issue). 3 choices each.
6. **Generate death messages** — 3 universal PR messages + 5-8 PR-specific ones
7. **Output** — `TRAIL_DATA:` JSON block with `trailTheme: "pr"` and `winDestination: "The Merge Frontier"`

Trail name derived from PR title: "Add JWT auth" → "The JWT Auth Trail"

Landmark type mapping:
| Change Rank | Landmark |
|---|---|
| 1 (core logic) | mountain |
| 2 (breaking API) | river |
| 3 (data model) | desert |
| 4 (non-breaking API) | forest |
| 5 (utility) | camp |
| 6 (tests) | town |
| 7 (config) | camp |
| First stop always | town |
