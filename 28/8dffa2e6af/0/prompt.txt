## File 1: `agents/pr-quiz-extractor.md`

**Pattern:** Follows `agents/trail-data-extractor.md` structure
**Model:** sonnet
**Tools:** Read, Grep, Glob, Bash(git:*), Bash(gh:*)

**Input:** Full PR diff text + PR metadata (title, description, author, number, URL)

**Algorithm:**
1. Group changed files into logical sections (same grouping algorithm as `commands/diff-review.md` Step 2 — by feature first, then by layer)
2. Rank sections by importance (1-7 scale from `skills/narration/references/ranking.md`)
3. Select top 4-8 sections as **trail stops**, each with:
   - Trail-themed location name (e.g., "Auth Pass", "Schema Ridge")
   - Most representative 15-25 line diff hunk as the code snippet
   - TTS-friendly narration explaining WHY the changes matter
   - Mapped `landmarkType` (schema changes = river, core logic = mountain, config = camp)
4. Generate 4-8 **quiz events** based on actual PR changes:
   - Question types from `agents/quiz-generator.md`: why, what-if, trace, connect, spot-the-issue
   - Each event mapped to a trail event type (weather/river/encounter/misfortune)
   - 3 choices per event with explanations
   - Each event tied to one of 4 party member concepts
5. Select 4 **party members** — the 4 most important concepts/areas in the PR
6. Generate 8+ **death messages** — PR-specific humor based on actual changes
7. Derive **trail name** from PR title (e.g., "The Authentication Trail")

**Output:** `TRAIL_DATA:` JSON block matching the format in `skills/visualization/SKILL.md`, with added `prMeta` field:
```json
{
  "prMeta": { "number": 42, "title": "...", "author": "...", "url": "...", "repo": "org/repo" }
}
```

## File 3: `.claude/commands/pr-quiz.md` (+ mirror `commands/pr-quiz.md`)

**Pattern:** Follows `commands/fly-visual.md` structure
**Model:** opus
**Argument-hint:** `<PR# | URL> [--post-proof]`
**Allowed-tools:** Same as fly-visual + Bash(gh:*) for PR operations

### Command Flow

**Step 1: Parse Arguments and Fetch PR Data**
- Extract PR number from input (number, URL with `/pull/`, or error if missing)
- Fetch PR context: `gh pr view <number> --json title,body,author,number,url`
- Fetch diff: `gh pr diff <number>`
- Validate: non-empty diff, PR is open

**Step 2: Extract Trail Data via Agent**
- Spawn `pr-quiz-extractor` agent (via Task tool, subagent_type: general-purpose)
- Input: diff text + PR metadata
- Output: TRAIL_DATA JSON
- Validate: has 4-8 stops, 4-8 events, 4 party members

**Step 3: Generate the HTML Game via `/playground`**
- Invoke the `playground` skill (via Skill tool)
- Provide the playground with the full game specification:
  - The TRAIL_DATA JSON to embed
  - Game spec inherited from fly-visual (canvas rendering, pixel art, state machine, scoring, keyboard support)
  - Title: "O'REORG TRAIL — PR Review Edition" with PR title as subtitle
  - The proof button spec from `skills/visualization/references/pr-quiz-proof.md`
  - The "prompt output" panel = proof/score card with Copy button
- The playground generates a self-contained HTML game file

**Step 4: Write and Open**
- Write to `.diff-review/pr-quiz-{PR_NUMBER}.html`
- Open in browser: `open .diff-review/pr-quiz-{PR_NUMBER}.html`
- TTS announcement via `scripts/tts.sh`

**Step 5: Terminal Feedback and Proof Handling**
- Display PR info summary + "Paste proof here when done, or type 'done' to exit"
- When user pastes proof (detected by `## PR Understanding Proof` header):
  - Parse and display formatted summary
  - Ask: "Post this as a comment on PR #{number}? (yes/no)"
  - If yes: `gh pr comment {number} --body "{proof}"`
- When user types `done`: exit without posting
